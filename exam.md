# 📚 ЭКЗАМЕН №1: Django MVT — «BookHouse»

**Предмет:** Django Template, Static, Media, CRUD (FBV)  
**Проект:** BookHouse — онлайн-библиотека  
**Время на выполнение:** 2 часа  
**Максимальный балл:** 100 баллов  

---

## 🎯 УСЛОВИЕ ЗАДАЧИ
Ваша задача — создать небольшое веб-приложение на Django под названием **BookHouse** для управления информацией о книгах. Проект должен быть реализован строго с использованием **Function Based Views (FBV)** и архитектуры **MVT**.

### Основные темы:
* **Template** (Наследование, контекст)
* **Static** (Подключение стилей)
* **Media** (Работа с изображениями)
* **CRUD** (Create, Read, Update, Delete)
* **Redirect** (Перенаправление после действий)

---

## 🛠 МОДЕЛИ ДАННЫХ

В проекте должно быть 4 модели со следующими полями:

### 1. Category (Категория)
* `name` — Название категории
* `description` — Краткая информация

### 2. Author (Автор)
* `full_name` — Полное имя
* `email` — Электронная почта
* `photo` — Фотография автора (**Media**)
* `bio` — Биография

### 3. Publisher (Издательство)
* `name` — Название
* `address` — Адрес
* `phone` — Номер телефона
* `logo` — Логотип (**Media**)

### 4. Book (Книга)
* `title` — Название книги
* `category` — Категория (**ForeignKey**)
* `author` — Автор (**ForeignKey**)
* `publisher` — Издательство (**ForeignKey**)
* `price` — Цена
* `pages` — Количество страниц
* `cover` — Изображение обложки (**Media**)
* `description` — Описание книги

---

## 📋 ТРЕБОВАНИЯ К ФУНКЦИОНАЛУ (CRUD)

Для **каждой** из 4-х моделей необходимо реализовать полный цикл CRUD через функции (FBV) в `views.py`:

1.  **Create:** Форма добавления нового объекта.
2.  **View/List:** Страница со списком всех объектов.
3.  **Detail:** Страница с подробной информацией об одном конкретном объекте.
4.  **Update:** Форма редактирования существующих данных.
5.  **Delete:** Функция удаления объекта.

---

## 💻 ШАБЛОНЫ (TEMPLATES) И ОФОРМЛЕНИЕ

### Наследование шаблонов
* Создать файл `base.html`, от которого наследуются все остальные страницы.
* **В base.html обязательно должны быть:** Название сайта, навигационное меню (ссылки на Home, Categories, Authors, Publishers, Books).

### Static файлы
* Подключить CSS-файл.
* Стилизовать меню, кнопки и списки. Сайт должен выглядеть аккуратно.

### Media файлы
* Настроить `MEDIA_URL` и `MEDIA_ROOT`.
* Отображать фотографии и обложки. Если изображения нет, выводить соответствующий текст.

### Redirect
* После успешного **создания**, **редактирования** или **удаления** записи, пользователь должен быть автоматически перенаправлен на нужную страницу (например, к списку).

---

## 🚀 ПОРЯДОК ВЫПОЛНЕНИЯ

1.  Создать Django проект и приложение.
2.  Описать 4 модели и настроить связи между ними.
3.  Выполнить миграции (`makemigrations`, `migrate`).
4.  Настроить папки `static` и `media` в `settings.py` и `urls.py`.
5.  Создать базовый шаблон `base.html` и дочерние шаблоны.
6.  Написать FBV функции для CRUD операций каждой модели.
7.  Настроить маршруты в `urls.py`.
8.  Завершить оформление сайта.

---

## ⚠️ ВАЖНЫЕ ТРЕБОВАНИЯ
* Использовать **только FBV** (никаких Class Based Views).
* Обязательно использование **Template Inheritance** (наследование).
* Наличие **Static** и **Media** файлов.
* Корректное использование **Redirect** после действий.
* Сайт должен работать без логических ошибок, имена должны быть понятными.
